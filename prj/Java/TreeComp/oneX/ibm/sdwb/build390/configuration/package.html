package for configuration classes.
These are classes used to maintain configuration choices.  Configuration switches should
only be used for behaviour that is not intended to change on a regular basis and that only
an administrator should be able to modify..  For example, if a given site likes to reuse PTF
numbers when rebuilding PTFs, that is a site policy that would require the client to behave
differently.  Also, this is not something an individual user should be able to decide or change.
Buildtype, on the other hand, could change every time you run a build. Also, every user
could have a different buildtype they wish to use.  So this should not be a configuration switch.
